# Script Task

<SimpleFlowDiagram label="Script Task" icon="script" client:load />

Evaluates an inline [FEEL](https://www.omg.org/spec/DMN/) expression and stores the result as process variables — no external worker or HTTP call needed. The engine runs the script synchronously before advancing the token.

## Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| ID | string | yes | Unique element identifier |
| Name | string | yes | Display label |
| Script | FEEL expression | yes | Expression to evaluate |
| Result Variable | string | conditional | Variable name for scalar output (omit when using context output) |

## Output Modes

### Context output

When the expression returns a context literal `{ key: value, ... }`, each key is stored as a separate process variable:

```feel
{ fee: amount * 0.05, tier: if amount > 1000 then "premium" else "standard" }
```

Both `fee` and `tier` become available to subsequent elements.

### Scalar output

When the expression returns a single value, set **Result Variable** to name where it is stored:

```feel
amount + shipping
```

If `resultVariable` is `totalCost`, the value is stored as the variable `totalCost`.

## FEEL Quick Reference

| Concept | Syntax |
|---------|--------|
| Equality | `=` (not `==`) |
| Inequality | `!=` |
| Boolean operators | `and`, `or`, `not(expr)` |
| String literals | `"double quotes"` |
| Context literal | `{ key: expr, key2: expr2 }` |
| Conditional | `if condition then value1 else value2` |
| List length | `count(myList)` |
| List access | `myList[1]` (1-based index) |
| Substring | `substring("hello", 1, 3)` → `"hel"` |

## XML Example

```xml
<bpmn:scriptTask id="calc_fee" name="Calculate Fee" scriptFormat="FEEL">
  <bpmn:script>
    { fee: amount * 0.05, tier: if amount > 1000 then "premium" else "standard" }
  </bpmn:script>
  <bpmn:incoming>flow_to_calc</bpmn:incoming>
  <bpmn:outgoing>flow_to_next</bpmn:outgoing>
</bpmn:scriptTask>
```

```xml
<!-- Scalar output with result variable -->
<bpmn:scriptTask id="sum_total" name="Sum Total" scriptFormat="FEEL"
  conduit:resultVariable="totalCost">
  <bpmn:script>amount + shipping</bpmn:script>
  <bpmn:incoming>flow_in</bpmn:incoming>
  <bpmn:outgoing>flow_out</bpmn:outgoing>
</bpmn:scriptTask>
```

## Notes

Script tasks execute synchronously inside the engine — they are not suitable for I/O-bound work (HTTP calls, database queries). For those use cases, use a [Service Task](/docs/elements/service-task/) instead.

## Cookbook

Script tasks have no mutation, so every "update a variable" idiom is **read the current value, return the new value**. The engine writes the result back to the instance — see [Variables & Scopes](/docs/concepts/variables/) for where it lands.

The return value shape decides what happens:

- **Context literal** `{ a: …, b: … }` — every key becomes a variable; `Result Variable` is ignored.
- **Scalar** (number, string, boolean, list) — written to `Result Variable` if set; dropped silently otherwise.

### Counter

Increment a variable by one each time this task runs.

```feel
{ counter: (if counter = null then 0 else counter) + 1 }
```

First run: `counter` is missing → defaulted to `0` → returns `1`. Subsequent runs increment from the stored value.

### Scalar counter

Same effect, single variable, scalar return shape. Set `Result Variable = counter` on the task.

```feel
(if counter = null then 0 else counter) + 1
```

### Accumulator

Sum a stream of inbound amounts.

```feel
{ total: (if total = null then 0 else total) + amount }
```

### Conditional counter

Only increment when something is true; pass through otherwise.

```feel
{ retries: if lastResult = "failure"
           then (if retries = null then 0 else retries) + 1
           else (if retries = null then 0 else retries) }
```

### Two outputs from one task

A context literal writes every key. Use it to set multiple related variables atomically.

```feel
{
  counter:    (if counter = null then 0 else counter) + 1,
  lastSeenAt: now()
}
```

### List append

Append an item to a list variable, initialising it the first time.

```feel
{ items: append(if items = null then [] else items, newItem) }
```

### Reset to a default

```feel
{ retries: 0 }
```

A flat literal writes `retries = 0` unconditionally. Use it after a successful retry to clear the counter.

### Aggregated stats

Build a nested structure as one variable.

```feel
{ stats:
    {
      count: count(orders),
      total: sum(orders),
      mean:  mean(orders)
    }
}
```

For more patterns (deep property access, null-safe accessors, gateway conditions) see the [Expressions Cookbook](/docs/concepts/expressions/) and the [FEEL Reference](/docs/decision-tables/feel-reference/).
